# How JS Engine Works

The JS engine reads, interprets, and executes JavaScript code.

## Execution Pipeline

| Step | Phase | Description |
|------|-------|-------------|
| 1 | **Tokenizing** (Lexical Analysis) | JS engine breaks raw code into small meaningful chunks called **tokens** |
| 2 | **Parsing** (Syntax Analysis) | Presents grammatical structure of code; syntax errors are caught here |
| 3 | **AST** (Abstract Syntax Tree) | Tokens are converted into a structured tree representing logic. Engine identifies keywords, identifiers, operators, values, punctuations |
| 4 | **Interpreter** (Ignition) | Creates lightweight **bytecode** from AST and starts executing immediately |
| 5 | **Execution** (Runtime with Call Stack) | **Cold code** (simple code that doesn't require optimization) is executed here |
| 6 | **Profiler + JIT Optimization** | **Hot code** (nested loops, callbacks, promises needing optimization) is identified and compiled. **TurboFan** compiler optimizes hot code and converts it to optimized bytecode |
| 7 | **Garbage Collector** | Removes unused/unreachable objects automatically. Frees memory, prevents leaks, keeps performance efficient |
| 8 | **Machine Code Execution** | The final optimized code is executed directly by the CPU for best runtime performance |

## How to View Bytecode

```bash
node --print-bytecode test.js
```

## Code Type Comparison

| Code Type | Definition | Example |
|-----------|------------|---------|
| **Source Code** | Human-written code using IDE (human understandable) | `console.log("Hello");` |
| **Bytecode** | Intermediate code created by JS engine (not human readable) | Generated by Ignition from AST |
| **Binary Code** | Machine code in 1s and 0s understood by CPU/chips | Final CPU-executable output |
