# JavaScript Introduction

## JavaScript — Key Features

| Feature | Description |
|---------|-------------|
| **Purpose** | Governs the interactive aspect of websites |
| **Paradigms** | Supports object-oriented, functional, and event-driven programming styles |
| **DOM Manipulation** | Manipulates DOM at runtime, making website behavior dynamic |
| **Typing** | Dynamically typed — no type declaration needed during variable assignment |
| **Execution** | Interpreted language with **JIT compilation** at runtime |
| **Engine** | Uses **V8 Engine** to convert program statements into machine code |
| **Async I/O** | Supports asynchronous I/O, efficient for API calls and file handling |
| **Ecosystem** | Massive ecosystem of libraries |

## Java vs JavaScript

| Aspect | Java / TypeScript | JavaScript |
|--------|-------------------|------------|
| **Typing** | Statically typed — requires data type declaration during variable assignment | Dynamically typed — no type declaration needed during variable assignment |

## Compilation vs Interpretation

| Aspect | Interpretation | Compilation |
|--------|---------------|-------------|
| **Process** | Line-by-line conversion to machine code | Conversion to machine code in one go |

## Node.js

| Aspect | Details |
|--------|---------|
| **What it is** | Open-source, cross-platform JavaScript runtime environment |
| **Formula** | JavaScript + V8 Engine + Server-Side Capabilities |
| **History** | Originally JS ran only in browsers (frontend); Node.js enables server-side execution |
| **Key Feature** | **Non-blocking I/O** — handles multiple requests asynchronously without waiting |
| **Use Cases** | Web servers, APIs, Chat Apps, CLI tools, IoT, Streaming Apps, Microservices |
