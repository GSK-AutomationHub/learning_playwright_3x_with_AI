JavaScript Null vs Undefined

In JavaScript, both null and undefined represent a lack of value, but they are used in entirely different contexts.undefined means a variable has been declared but has not yet been assigned a value. The JavaScript engine sets this automatically.null is an intentional assignment representing the deliberate absence of any object value. You explicitly set a variable to null to mark it as empty

Modern Handling: The Nullish Coalescing Operator (??)If you want to provide a default fallback value for both null and undefined without accidentally catching other falsy values (like 0 or ""), use the nullish coalescing operator:javascriptlet score = 0; // Valid falsy number
let defaultScore = score ?? 10; // returns 0

let enteredName = null;
let displayName = enteredName ?? "Guest"; // returns "Guest"
