JavaScript, identifiers and literals

JavaScript, identifiers and literals are the core building blocks of code syntax. An identifier is the user-defined name given to a variable, function, or property. A literal is a fixed, concrete value written directly into your script.Think of an identifier as a labeled container, and a literal as the actual item stored inside i

Comparison OverviewFeatureIdentifierLiteralPurposeNames variables, functions, and properties.Represents actual data values directly.FlexibilityPoints to values that can change.Represents a fixed, unchangeable value.QuotationNever wrapped in quotes.Strings are wrapped in quotes; others use syntax.ExampletotalAmount, getUser, _cache42, "Hello", true, [1, 2, 3]

What is an Identifier?An identifier is a sequence of characters that acts as a custom name in your code. You use them to reference items later.Naming RulesMust start with a letter, underscore (_), or dollar sign ($).Cannot start with a digit (0-9).Can contain subsequent letters, digits, underscores, or dollar signs.Are completely case-sensitive (user and User are different).Cannot be a reserved JavaScript keyword like let, function, or class

// Valid identifiers
let firstName = "Alex";  // firstName is the identifier
const $price = 99;       // $price is the identifier
function _calculate() {} // _calculate is the identifier

// Invalid identifiers
let 1user = "Mark";      // Error: Cannot start with a digit
let user-name = "Mark";  // Error: Hyphens are not allowed
let const = 10;          // Error: 'const' is a reserved keyword

What is a Literal?A literal is data provided directly in the source code. JavaScript uses different types of literals to represent various data types.Common Types of LiteralsNumeric Literals: Regular numbers (10, 3.14).String Literals: Text wrapped in quotes ('hello', "world").Boolean Literals: The direct keywords true or false.Array Literals: Collections enclosed in square brackets ([1, 2, 3]).Object Literals: Key-value pairs enclosed in curly braces ({ age: 25 }).Template Literals: Strings using backticks for expression evaluation (`Hello ${name}`)

let count = 5;                  // 5 is a numeric literal
let greeting = "Hello";         // "Hello" is a string literal
