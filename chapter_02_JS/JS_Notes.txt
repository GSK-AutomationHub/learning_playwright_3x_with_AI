JavaScript
 -  Governs interactive aspect of any website 
 -  Supports object-oriented, functional, and event-driven programming styles
 -  Manipulate DOM runtime making website behavior dynamic
 -  Dynamically typed, so data type declaration not require during variable assignment
 -  Its interpreted language with JIT compilation at runtime.
 -  Uses v8 JS Engine to covert program statements into machine code
 -  Support Asynchronous I/O, so efficient in performing API calls and file handling.
 -  Has massive ecosystem of libraries

Java. vs JavaScript
 -  JS is dynamically typed so does not follow rule of data type declaration during variable assignment
 -  Java & TS are not dynamically typed & requires data type to be mentioned during variable assignment

Compilation vs Interpretation
 -  Interpretation - line by line conversion to machine code
 -  Compilation - conversion to machine code in one go

Node.js
 -  JavaScript runtime environment which is open source & cross-platform 
 -  Its JavaScript + V8 Engine + Server-Side Capabilities
 -  Originally JS designed to run in web browsers (frontend)only, now with Node.js it can be run on servers.
 -  Main feature - Non blocking I/O - handles multiple request asynchronously w/o wait
 -  Commonly used in - Webservers, APIs, Chat Apps, CLI tools, IoT, Steaming Apps, Microservices
